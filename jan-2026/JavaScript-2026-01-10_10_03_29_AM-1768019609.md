## Interview Question: JavaScript Hoisting & `typeof`

### Question

What will be the output of the following JavaScript code, and why?

```js
console.log(typeof myVar);
console.log(typeof test);
console.log(typeof fatArrow);

var myVar = 'user';
function test() {
  console.log('Function executed');
}
var fatArrow = () => {};

console.log(typeof myVar);
console.log(typeof test);
console.log(typeof fatArrow);
```

---

### Answer

#### Output

```js
undefined
function
undefined

string
function
function
```

---

### Explanation (Key Points)

- **`var` declarations are hoisted** but initialized as `undefined`.

  - `typeof myVar` → `undefined`
  - `typeof fatArrow` → `undefined`

- **Function declarations are fully hoisted** (both definition and body).

  - `typeof test` → `function`

- After execution reaches the assignments:

  - `myVar` holds a string → `string`
  - `fatArrow` is assigned a function → `function`

---

### Summary Table

| Identifier | Before Declaration | After Declaration |
| ---------- | ------------------ | ----------------- |
| `myVar`    | `undefined`        | `string`          |
| `test`     | `function`         | `function`        |
| `fatArrow` | `undefined`        | `function`        |

---

### Interview Insight

> Function declarations are hoisted completely, while variables declared with `var` are hoisted without initialization.
> Arrow functions behave like variables, **not** function declarations.
